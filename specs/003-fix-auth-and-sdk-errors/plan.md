# Implementation Plan: 인증 함수 내보내기 오류 및 Kakao SDK 401 에러 수정

**Branch**: `003-fix-auth-and-sdk-errors` | **Date**: 2026-03-19 | **Spec**: [specs/003-fix-auth-and-sdk-errors/spec.md]
**Input**: Feature specification from `/specs/003-fix-auth-and-sdk-errors/spec.md`

## Summary

이 계획은 현재 애플리케이션의 실행을 방해하는 두 가지 치명적인 오류를 해결하는 것을 목표로 합니다. 첫째, `src/api/auth.ts`와 `MapPage.tsx` 간의 인증 상태 구독 함수 이름 불일치를 수정하여 런타임 SyntaxError를 해결합니다. 둘째, 카카오 맵 SDK 로드 시 환경 변수를 올바르게 주입하여 401 Unauthorized 에러를 해결하고 지도가 정상적으로 렌더링되도록 합니다.

## Technical Context

**Language/Version**: TypeScript 5.x, Node.js 20.x  
**Primary Dependencies**: React 18+, react-kakao-maps-sdk, Firebase SDK 10+  
**Storage**: Firebase Auth  
**Testing**: Vitest (Unit), Playwright (E2E)  
**Target Platform**: Web (Desktop/Mobile Responsive)  
**Project Type**: Web Application  
**Performance Goals**: 지도 로딩 지연 최소화  
**Constraints**: Kakao Map API 키 보안 준수 (VITE_ 접두사 확인)

**NEEDS CLARIFICATION**:
- [ ] `src/api/auth.ts`에서 실제로 내보내고 있는 인증 구독 함수의 정확한 이름 확인 필요.
- [ ] 카카오 SDK 로딩이 `index.html`에서 정적으로 수행되는지, 아니면 별도의 훅이나 유틸리티에서 동적으로 수행되는지 확인 필요.
- [ ] `.env` 파일에 `VITE_KAKAO_JAVASCRIPT_KEY`가 실제로 정의되어 있고 값이 유효한지 확인 필요.
