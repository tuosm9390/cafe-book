# Feature Specification: 지도 초기화 및 레이아웃 수정

**Feature Branch**: `005-fix-map-initialization`  
**Created**: 2026-03-19  
**Status**: Draft  
**Input**: User description: "로그인 이후 메인페이지 이동 시 사이드바가 화면 중앙에 위치하고 있고 지도 맵은 아무것도 표시가 안되고 있어. 현재 내 위치를 탐색하고 위치를 중심으로 지도 표시되게 해줘."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 지도 초기 위치 자동 설정 (Priority: P1)

사용자가 서비스에 로그인하여 메인 페이지(지도 페이지)에 접속했을 때, 시스템은 사용자의 현재 위치를 파악하여 지도의 중심을 해당 위치로 이동시킨다.

**Why this priority**: 사용자가 주변 카페 정보를 즉시 확인할 수 있도록 하여 서비스의 유용성을 극대화한다.

**Independent Test**: 브라우저의 위치 정보 권한을 활성화한 상태에서 지도 페이지에 접속했을 때, 사용자의 현재 위치를 나타내는 지점이 중심에 오는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 로그인한 상태이고 위치 권한이 허용됨, **When** 메인 페이지에 진입함, **Then** 지도가 사용자의 현재 위도/경도를 중심으로 렌더링됨

---

### User Story 2 - 사이드바 레이아웃 정상화 (Priority: P1)

사용자가 대화면(데스크톱)에서 접속했을 때, 사이드바는 화면 중앙이 아닌 좌측에 고정되어야 하며 지도는 나머지 공간을 가득 채워야 한다.

**Why this priority**: 비정상적인 UI 배치는 서비스 신뢰도를 떨어뜨리고 지도 사용을 방해한다.

**Independent Test**: 데스크톱 해상도에서 페이지를 열었을 때 사이드바가 좌측에 위치하고 지도가 우측 나머지 공간을 차지하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 데스크톱 브라우저 환경, **When** 페이지 로드 시, **Then** 사이드바는 좌측 80px(또는 지정된 너비)에 위치하고 지도는 우측 전체 영역을 차지함

---

### Edge Cases

- **위치 권한 거부 시**: 브라우저에서 위치 정보 제공을 거부한 경우, 사전에 정의된 기본 위치인 강남역(37.4979, 127.0276)을 중심으로 지도를 표시한다.
- **Geolocation API 미지원 브라우저**: 구형 브라우저 등에서 위치 정보를 가져올 수 없는 경우 에러 메시지 없이 기본 위치로 폴백한다.
- **위치 정보 획득 지연**: 위치 정보를 가져오는 동안 로딩 상태를 표시하거나 기본 위치를 먼저 보여준 후 업데이트한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 페이지 로드 시 브라우저 Geolocation API를 사용하여 현재 좌표를 요청해야 한다.
- **FR-002**: 위치 정보를 성공적으로 가져오면 지도의 중심 좌표를 업데이트해야 한다.
- **FR-003**: 위치 권한 거부 시 표시할 기본 위치는 강남역(위도: 37.4979, 경도: 127.0276)으로 설정한다.
- **FR-004**: 사이드바가 중앙에 배치되는 CSS 이슈를 해결해야 한다. (body 태그의 `place-content: center` 등 레이아웃 간섭 제거)
- **FR-005**: 초기 지도 확대 레벨(level)은 5로 설정한다.

### Key Entities

- **GeolocationState**: 사용자의 현재 위도(latitude), 경도(longitude) 및 로딩/에러 상태.
- **MapConfig**: 지도의 중심 좌표 및 확대 레벨 설정값.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 페이지 로드 후 2초 이내에 지도가 렌더링되고 사용자의 현재 위치(또는 기본 위치)를 중심으로 표시된다. 로딩 중에는 로딩 스피너나 메시지가 중앙에 표시되어야 한다.
- **SC-002**: 데스크톱 해상도(768px 이상)에서 사이드바가 화면 좌측 끝에 고정(Fixed/Flex 좌측 배치)되며 지도가 가로 전체의 나머지 영역을 100% 차지한다.
