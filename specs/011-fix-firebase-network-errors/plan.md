## Summary

`src/api/firebase.ts`에서 발생하는 TypeScript 타입 에러(`useFetchStreams` 미정의)를 해결하고, 특정 네트워크 환경에서 Firestore 클라이언트가 오프라인으로 고착되는 현상을 방지하기 위해 HTTP Long Polling 설정을 강제합니다. 또한 오염된 IndexedDB 캐시를 초기화할 수 있는 복구 로직을 복원합니다.

## Technical Context

**Language/Version**: TypeScript 5.x, Node.js 20.x  
**Primary Dependencies**: React 18+, Firebase SDK 10.13.0+  
**Storage**: Firebase Firestore  
**Testing**: Vitest (Unit), Playwright (E2E)  
**Constraints**: Firebase SDK v10+ 모듈형 스타일 유지, `FirestoreSettings` 타입 준수.
