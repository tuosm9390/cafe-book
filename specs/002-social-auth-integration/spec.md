# Feature Specification: 소셜 로그인 연동 및 로그인 UI 구현

**Feature Branch**: `002-social-auth-integration`  
**Created**: 2026-03-19  
**Status**: Draft  
**Input**: User description: "카카오, 구글 로그인 기능 연동해줘. 그리고 첫페이지는 로그인페이지가 나오도록 리다이렉트해줘. 로그인화면에는 오른쪽으로 3/1정도 화면에 사이드바로 구성하고 로그인 버튼과 회원가입버튼 표시해줘. 회원가입버튼 클릭시에는 각각버튼이 카카오회원가입, 구글회원가입 버튼으로 바뀌게해줘."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 소셜 계정으로 로그인 (Priority: P1)

사용자는 별도의 회원가입 과정 없이 기존의 카카오 또는 구글 계정을 사용하여 서비스에 접속할 수 있다.

**Why this priority**: 사용자 접근성을 높이고 회원가입 허들을 낮추어 서비스 이용률을 증대시킨다.

**Independent Test**: 로그인 페이지에서 카카오/구글 로그인 버튼을 클릭했을 때 각 제공자의 인증 화면으로 정상적으로 이동하고, 인증 완료 후 서비스 메인 화면으로 리다이렉트되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 로그아웃 상태에서 서비스 접속 시, **When** 카카오 로그인 버튼을 클릭하면, **Then** 카카오 인증 페이지가 열린다.
2. **Given** 카카오 인증 완료 후, **When** 서비스로 돌아오면, **Then** 로그인 상태가 유지되며 메인 페이지(지도 페이지)로 이동한다.
3. **Given** 로그아웃 상태에서 서비스 접속 시, **When** 구글 로그인 버튼을 클릭하면, **Then** 구글 인증 페이지가 열린다.

---

### User Story 2 - 로그인 페이지 리다이렉트 및 UI (Priority: P1)

비인증 사용자가 서비스에 처음 접속했을 때, 시스템은 자동으로 로그인 페이지로 리다이렉트하며 최적화된 레이아웃을 제공한다.

**Why this priority**: 서비스 보안 및 사용자 식별을 위한 필수 과정이며, 첫 인상을 결정하는 중요한 UI 요소이다.

**Independent Test**: 브라우저에서 서비스 루트 URL 접속 시 `/login` 경로로 자동 리다이렉트되는지 확인하고, 화면 우측에 1/3 비율의 로그인 사이드바가 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 비인증 사용자가 `index.html` 또는 `/` 경로로 접속 시, **When** 페이지가 로드되면, **Then** `/login` 페이지로 자동 리다이렉트된다.
2. **Given** 로그인 페이지에서, **When** 화면이 렌더링되면, **Then** 우측 1/3 영역에 로그인/회원가입 버튼이 포함된 사이드바가 표시된다.

---

### User Story 3 - 회원가입 버튼 전환 (Priority: P2)

사용자가 회원가입을 원할 경우, 기존 로그인 버튼들이 소셜 회원가입 전용 버튼으로 직관적으로 변경된다.

**Why this priority**: 화면 공간을 효율적으로 사용하면서도 사용자의 의도(회원가입)에 맞는 명확한 행동 지침을 제공한다.

**Independent Test**: 로그인 사이드바에서 '회원가입' 버튼을 클릭했을 때 버튼 텍스트가 '카카오 회원가입', '구글 회원가입'으로 즉시 변경되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 로그인 페이지 사이드바에서, **When** '회원가입' 버튼을 클릭하면, **Then** 기존의 '로그인' 버튼들이 사라지거나 '카카오 회원가입', '구글 회원가입' 버튼으로 교체된다.

---

### Edge Cases

- **인증 취소**: 사용자가 카카오/구글 인증 화면에서 '취소'를 눌렀을 때, 시스템은 에러 메시지를 표시하거나 다시 로그인 페이지를 유지해야 한다.
- **네트워크 오류**: 소셜 로그인 API 호출 중 네트워크 장애가 발생할 경우, 적절한 안내 메시지를 제공해야 한다.
- **이미 로그인된 상태**: 로그인된 사용자가 `/login` 경로로 직접 접속할 경우, 메인 페이지로 다시 리다이렉트해야 한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 비인증 사용자의 모든 요청을 `/login` 페이지로 리다이렉트해야 한다.
- **FR-002**: 로그인 페이지는 화면 우측 약 33% 영역을 차지하는 사이드바 레이아웃을 가져야 한다.
- **FR-003**: 사이드바 초기 상태에는 '로그인' 섹션(카카오, 구글 로그인 버튼)과 '회원가입' 버튼이 포함되어야 한다.
- **FR-004**: '회원가입' 버튼 클릭 시, 소셜 로그인 버튼들은 각각 '카카오 회원가입', '구글 회원가입' 텍스트로 전환되어야 한다.
- **FR-005**: 카카오/구글 OAuth2 인증 기능을 통합하여 성공적인 인증 후 사용자 세션을 생성해야 한다.

### Key Entities

- **User**: 소셜 인증을 통해 식별된 사용자 정보 (ID, 이메일, 프로필 이미지 등)
- **Auth Provider**: 인증 서비스를 제공하는 외부 플랫폼 (Kakao, Google)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 비인증 사용자 접속 시 0.5초 이내에 로그인 페이지로 리다이렉트 완료.
- **SC-002**: 회원가입 버튼 클릭 시 버튼 전환 애니메이션 또는 상태 변경이 즉각적으로(100ms 이내) 수행됨.
- **SC-003**: 소셜 로그인 완료 후 메인 페이지 로딩까지의 총 시간이 3초 이내여야 함.
- **SC-004**: 다양한 화면 해상도에서 로그인 사이드바가 항상 우측 1/3 영역을 유지해야 함(반응형 고려).
