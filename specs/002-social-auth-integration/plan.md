# Implementation Plan: 소셜 로그인 연동 및 로그인 UI 구현

**Branch**: `002-social-auth-integration` | **Date**: 2026-03-19 | **Spec**: [specs/002-social-auth-integration/spec.md]
**Input**: Feature specification from `/specs/002-social-auth-integration/spec.md`

## Summary

이 기능은 카카오 및 구글 소셜 로그인을 통합하여 사용자 인증 허들을 낮추고, 비인증 사용자를 위한 직관적인 로그인 전용 페이지를 구축하는 것을 목표로 합니다. 우측 1/3 영역을 차지하는 사이드바 레이아웃을 통해 세련된 UI를 제공하며, 회원가입 의도에 따라 버튼 텍스트가 동적으로 전환되는 UX를 구현합니다. 기술적으로는 Firebase Auth를 활용하여 소셜 로그인을 처리하고, React Router를 통해 리다이렉션 로직을 관리합니다.

## Technical Context

**Language/Version**: TypeScript 5.x, Node.js 20.x  
**Primary Dependencies**: React 18+, Firebase SDK 10+, Tailwind CSS, Lucide React (icons), React Router v6  
**Storage**: Firebase Auth, Firestore (User Profile)  
**Testing**: Vitest (Unit), Playwright (E2E)  
**Target Platform**: Web (Desktop/Mobile Responsive)  
**Project Type**: Web Application  
**Performance Goals**: 리다이렉션 < 0.5s, 버튼 전환 < 0.1s  
**Constraints**: Firebase 프로젝트 설정 필요 (Kakao/Google provider), `.env` 보안 준수

**NEEDS CLARIFICATION**:
- [ ] 카카오 로그인의 경우 Firebase Auth에서 직접 지원하지 않으므로, 커스텀 인증(Firebase Functions 사용) 또는 별도의 라이브러리 연동 방식 결정 필요.
- [ ] 리다이렉션 로직을 `App.tsx` 최상단에서 처리할지, 별도의 `ProtectedRoute` 컴포넌트를 구성할지 결정 필요.
- [ ] 1/3 사이드바 디자인의 경우, 모바일 환경(Breakpoint < 768px)에서 전체 화면으로 전환할지 여부 결정 필요.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- [x] **I. Library-First**: Firebase Auth 관련 로직을 `src/api/auth.ts`로 분리하여 컴포넌트와 격리함.
- [x] **II. Mobile-First**: Tailwind CSS의 `md:w-1/3` 등을 활용하여 모바일 대응 레이아웃 설계.
- [x] **III. Hybrid Testing**: 소셜 로그인 연동(E2E) 및 버튼 텍스트 전환(Unit) 테스트 시나리오 식별됨.
- [x] **IV. Data Security**: API 키 및 소셜 앱 ID는 `.env`로 관리하며 Firebase Rules로 데이터 보호.
- [x] **V. Performance**: 리다이렉션 성능 목표(0.5s) 준수를 위한 코드 스플리팅 고려.
