# Implementation Plan: Social Authentication Configuration Fix

**Branch**: `004-fix-social-auth` | **Date**: 2026-03-19 | **Spec**: [specs/004-fix-social-auth/spec.md]
**Input**: Feature specification from `/specs/004-fix-social-auth/spec.md`

## Summary

이 계획은 사용자가 보고한 Firebase `CONFIGURATION_NOT_FOUND` (400) 오류를 해결하고, Google 및 Kakao 소셜 로그인 기능을 정상화하는 것을 목표로 합니다. 주요 기술적 접근은 `.env` 환경 변수가 Firebase 및 Kakao SDK 초기화 시점에 올바르게 주입되는지 검증하고, 특히 Kakao 로그인의 경우 Firebase Custom Token 인증 흐름을 완벽하게 구현하는 것입니다.

## Technical Context

**Language/Version**: TypeScript 5.x, Node.js 20.x  
**Primary Dependencies**: React 18+, Firebase SDK 10+, Kakao SDK  
**Storage**: Firebase Firestore, Firebase Auth  
**Testing**: Vitest (Unit), Playwright (E2E)  
**Target Platform**: Web (Desktop/Mobile Responsive)  
**Project Type**: Web Application  
**Performance Goals**: Auth process < 5s  
**Constraints**: Firebase API Key security (VITE_ prefix), Kakao API restrictions

**NEEDS CLARIFICATION**:
- [x] `src/api/firebase.ts`에서 `import.meta.env`를 사용하는 방식은 Vite 표준으로 확인됨. (연구 완료)
- [x] Kakao 로그인을 위해 Firebase Identity Platform의 커스텀 인증 또는 백엔드 토큰 교환 로직이 필수적임을 확인. (설계 반영)
- [x] 모든 환경 변수에 `VITE_` 접두사가 올바르게 사용되고 있음을 `.env` 조사를 통해 확인. (검증 완료)
