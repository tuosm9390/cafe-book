# Feature Specification: 카페 도감 초기 기능 (지도 및 관리 페이지)

**Feature Branch**: `001-cafe-map-admin`  
**Created**: 2026-03-19  
**Status**: Draft  
**Input**: User description: "새로운 프로젝트를 만드려고 해. 주제는 카페 도감이야. 우선 현재는 2개의 페이지가 필요해. 1. 지도 페이지 - 왼쪽 사이드바에는 카페의 목록을 보여주고 검색할 수 있어야 해. - 본 페이지는 카카오맵 api를 사용해서 지도로 보여줘야해. - 각 리스트에 코멘트를 작성하고, 별점을 체크할 수 있어야 해. 즐겨찾기 기능도 있어야 해. - 각 리스트를 선택할 때마다 지도에서는 선택한 아이템의 위치정보를 바탕으로 지도화면 중앙에 표시됨. 2. 관리 페이지 - 카페에 대한 정보를 입력하고 입력받은 정보로 지도 페이지에 표시되야해. - 리스트를 수정 및 삭제할 수 있어야 해."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 지도 기반 카페 탐색 및 검색 (Priority: P1)

사용자는 지도 페이지를 통해 등록된 카페 목록을 확인하고, 검색 기능을 통해 특정 카페를 찾을 수 있습니다. 또한, 목록에서 특정 카페를 선택하여 지도의 중심에서 해당 위치를 확인할 수 있습니다.

**Why this priority**: 프로젝트의 핵심 기능인 '지도 기반 정보 제공'을 달성하기 위해 가장 우선적으로 구현되어야 합니다.

**Independent Test**: 카페 데이터를 수동으로 주입했을 때, 목록에 올바르게 표시되고 검색 결과가 필터링되며, 선택 시 지도가 해당 좌표로 이동하는지 확인하여 테스트할 수 있습니다.

**Acceptance Scenarios**:

1. **Given** 카페 목록에 "카페 A", "카페 B"가 있을 때, **When** 사용자가 검색창에 "카페 A"를 입력하면, **Then** 사이드바 목록에는 "카페 A"만 표시되어야 함.
2. **Given** 사이드바에서 특정 카페를 클릭했을 때, **When** 해당 카페의 위치 정보가 유효하다면, **Then** 지도의 중심점이 해당 위치로 즉시 이동하고 마커가 표시되어야 함.

---

### User Story 2 - 카페 데이터 관리 (Priority: P1)

관리자(또는 사용자)는 관리 페이지를 통해 카페의 이름, 위치 정보 등을 입력하여 등록할 수 있으며, 기존에 등록된 정보를 수정하거나 삭제할 수 있습니다.

**Why this priority**: 지도에 표시될 데이터를 생성하고 유지보수하기 위해 필수적인 관리 기능입니다.

**Independent Test**: 관리 페이지에서 폼을 통해 데이터를 입력하고 '등록' 버튼을 눌렀을 때, 데이터가 저장되고 목록 페이지에 즉시 반영되는지 확인하여 테스트할 수 있습니다.

**Acceptance Scenarios**:

1. **Given** 관리 페이지의 입력 폼에서, **When** 새로운 카페 이름과 위치 정보를 입력하고 등록하면, **Then** 지도 페이지의 목록과 지도상에 해당 카페가 나타나야 함.
2. **Given** 등록된 카페 리스트에서 특정 아이템의 '삭제' 버튼을 눌렀을 때, **When** 확인 절차를 거치면, **Then** 해당 카페 데이터가 영구적으로 제거되고 지도 및 목록에서 사라져야 함.

---

### User Story 3 - 카페 상호작용 (코멘트, 별점, 즐겨찾기) (Priority: P2)

사용자는 특정 카페에 대해 개인적인 코멘트를 남기고 별점을 매길 수 있으며, 자주 방문하거나 선호하는 카페를 즐겨찾기에 추가할 수 있습니다.

**Why this priority**: 단순한 정보 제공을 넘어 사용자가 직접 참여하고 개인화된 경험을 제공하기 위한 기능입니다.

**Independent Test**: 특정 카페 리스트 항목에서 코멘트를 작성하고 별점을 변경했을 때 해당 값이 저장되는지, 그리고 즐겨찾기 토글 시 상태가 유지되는지 확인하여 테스트할 수 있습니다.

**Acceptance Scenarios**:

1. **Given** 특정 카페 항목의 코멘트 입력란에서, **When** 내용을 작성하고 저장하면, **Then** 다음에 해당 카페를 조회할 때 작성한 내용이 그대로 표시되어야 함.
2. **Given** 카페 항목의 즐겨찾기 아이콘을 클릭했을 때, **When** 상태가 '활성'으로 변경되면, **Then** 해당 카페는 즐겨찾기 필터링 또는 별도의 표시를 통해 선호 카페임을 알 수 있어야 함.

---

### Edge Cases

- **좌표 정보 부재**: 카페 등록 시 위치 정보(위도, 경도)가 누락되거나 잘못된 경우 시스템은 이를 어떻게 처리하는가?
- **검색 결과 없음**: 사용자가 입력한 검색어와 일치하는 카페가 없을 때 사이드바에 어떤 피드백을 제공하는가?
- **데이터 충돌**: 동일한 이름이나 위치를 가진 카페를 중복 등록하려고 할 때 시스템의 대응 방식은?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 카카오맵 API를 연동하여 지도를 화면 전체에 렌더링해야 함.
- **FR-002**: 사이드바에는 현재 시스템에 등록된 모든 카페의 목록이 표시되어야 하며, 이름 기반 검색이 가능해야 함.
- **FR-003**: 사이드바 리스트에서 아이템을 선택할 경우 지도의 중심 좌표를 해당 아이템의 좌표로 갱신해야 함.
- **FR-004**: 각 카페 아이템은 코멘트 작성 기능과 별점 체크(1~5점) 기능을 포함해야 함.
- **FR-005**: 각 카페 아이템은 즐겨찾기(ON/OFF) 상태를 가질 수 있어야 함.
- **FR-006**: 관리 페이지는 카페 정보를 입력받는 폼(이름, 위치 등)을 제공하고 이를 **Firebase(Firestore)** 데이터베이스에 저장해야 함.
- **FR-007**: 관리 페이지는 등록된 카페 리스트를 보여주고 수정 및 삭제 기능을 제공해야 함. 보안을 위해 **인증된 관리자 계정**만 접근 가능해야 하며, 추후 일반 사용자 권한 확장성을 고려하여 설계함.
- **FR-008**: 작성된 코멘트와 별점 정보는 영구적으로 저장되어야 하며, **모든 사용자에게 공개**되어 조회 가능해야 함.

### Key Entities *(include if feature involves data)*

- **Cafe (카페)**: 이름, 주소, 좌표(위도, 경도), 등록일시 등을 포함하는 핵심 데이터 객체.
- **Interaction (상호작용)**: 특정 카페에 대한 사용자의 코멘트, 별점, 즐겨찾기 상태를 관리하는 데이터.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 카페 이름을 검색했을 때 0.5초 이내에 목록이 필터링되어야 함.
- **SC-002**: 리스트 아이템 클릭 시 지도가 끊김 없이(Smooth transition) 해당 위치로 이동하여 중앙에 배치되어야 함.
- **SC-003**: 새로운 카페 정보를 관리 페이지에서 등록한 후 지도 페이지로 이동했을 때 즉시 반영되어 있어야 함.
- **SC-004**: 사용자가 작성한 코멘트와 별점이 페이지 새로고침 후에도 정확하게 유지되어야 함.
